詳解

😁 （※翻訳者追記） いわゆるプログラムで言う所の、インストラクション（Instruction；命令）だ。つまりこれは、ジャンプ命令（ジャンプにも色々あるだろ。何がジャンプするんだ？）

  • （※翻訳者追記）リファクタリング指南
  • （※翻訳者追記）　　わたしがもっといい名前空間を考えてあげよう。　 GatchaGatchaGames.RPGMakerUnite.Engine.Instructions.Jump クラスだ。その方が適切だと頭の中で思っておけばよい
  • （※翻訳者追記）　　空間は作るものではなく、１つの大きな空間を切っていく（パーティションしていく）感覚だ。作るではなく、切ると表現することにしよう
  • （※翻訳者追記）　　理想的には名前空間 GatchaGatchaGames.RPGMakerUnite.Engine.Instructions を切る
  • （※翻訳者追記）　　理想的にはこのクラス名を単に Jump とする
  • （※翻訳者追記）　　プログラムで使う時はファイルの冒頭あたりで using TheInstruction = GatchaGatchaGames.RPGMakerUnite.Engine.Instructions; と、名前空間の別名を切る
  • （※翻訳者追記）　　プログラムでこのクラスを使う時は TheInstruction.Jump のように型指定する
  • （※翻訳者追記）　　名前空間は、大辞典を作るためにあるのではなく、チーム開発で作業分担できるように獲物をざっくり山分けしやすいように名付けるのだ。物の切れ目ではなく、作業の切れ目が見えなければいけない。細かく分類することに意味はない
